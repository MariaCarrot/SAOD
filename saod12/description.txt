По результатам 10 независимых запусков установлено, что алгоритм сортировки слиянием показывает существенно лучшую масштабируемость по сравнению с сортировкой вставками. 
Экспериментальные данные для merge sort хорошо согласуются с теоретической сложностью O(nlogn): при увеличении размера списка время выполнения и число операций растут умеренно и значительно медленнее квадратичного закона. 
Для insertion sort на случайных, отсортированных и почти отсортированных данных наблюдается квадратичный рост числа сравнений и времени выполнения, что соответствует сложности O(n^2).
Для отсортированного входа сортировка вставками не выполняет изменений указателей, поэтому значение метрики равно нулю. На логарифмической шкале такие точки не отображаются, поэтому для визуализации они были заменены минимальным положительным значением.
При этом для insertion sort набор данных Reverse показал линейное поведение, а Sorted — квадратичное. Это связано с особенностью реализации алгоритма. В альтернативной реализации набор данных Reverse показал бы квадратичное поведение, а Sorted — линейное. 
Из этого можно сделать вывод insertion sort эффективна только на благоприятных входах и плохо масштабируется, тогда как merge sort остаётся эффективной даже на больших размерах входных данных.
